Claude Code 初學者架構教學

本教學採同頁多層次設計：每章同時提供 90 秒總覽、執行路徑、設計取捨、失敗路徑與實作驗證，讓不同程度讀者都能在同一頁閱讀。

首頁 快速起步 啟動流程 指令系統 工具系統 回合引擎 服務與 MCP 狀態與任務 模式與權限 記憶系統 代理與擴充 提示系統 Hooks 與自動化 終端 UI

先選你的閱讀深度

本網站採「同頁多層次」架構：每章都從快讀到深挖，避免你在多頁切換中迷路。

章節閱讀骨架

所有章節遵循同一個閱讀順序，確保你在不同主題下也能維持一致節奏。

  1. 90 秒總覽：先定義這章解決的核心問題。
  2. 執行路徑：用流程圖與片段程式碼掌握主幹。
  3. 關鍵決策：理解設計選擇與代價。
  4. 替代方案取捨：知道為什麼不是另一種做法。
  5. 失敗路徑：先看錯誤，再看防護。
  6. 實作驗證：用最小驗證集確認你真的看懂。

我一定要照順序讀完 13 章嗎？

不用。先看首頁的角色路線快速上手，再補四階段路線，效率最高。

每章都要看 L1、L2、L3 嗎？

不必。建議先讀 L1 建立地圖，遇到工作需求時再下鑽 L2/L3。

我想直接改程式，先看哪裡？

先看「啟動流程、回合引擎、模式與權限」，這三章最能降低改動風險。
