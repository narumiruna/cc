Claude Code 初學者架構教學

本教學採同頁多層次設計：每章同時提供 90 秒總覽、執行路徑、設計取捨、失敗路徑與實作驗證，讓不同程度讀者都能在同一頁閱讀。

首頁 快速起步 啟動流程 指令系統 工具系統 回合引擎 服務與 MCP 狀態與任務 模式與權限 記憶系統 代理與擴充 提示系統 Hooks 與自動化 終端 UI

如何使用本教學

每一章採「同頁多層次」設計，以三種深度標籤區分內容：

L1 入門 L2 實務 L3 原理
  • L1 入門 — 90 秒即可讀完的概念總覽，適合初次接觸的讀者快速掌握全貌。
  • L2 實務 — 執行路徑、關鍵決策與程式碼片段，適合想動手實作的開發者。
  • L3 原理 — 設計取捨、替代方案比較與失敗路徑分析，適合追求深層理解的架構師。

建議先從 L1 通讀一遍建立心智圖，再依需求逐章深入 L2 與 L3。

每章結構

所有章節皆遵循統一模板，包含以下六個區塊：

  1. 90 秒總覽 — 一段話概述該章主題，讓你在最短時間建立認知框架。
  2. 執行路徑 — 以流程圖與虛擬碼呈現核心邏輯的運作順序。
  3. 關鍵決策 — 標註原始碼中重要的分支判斷與設計選擇。
  4. 替代方案取捨 — 比較不同方案的優缺點，理解為何選擇目前的做法。
  5. 失敗路徑 — 列出常見錯誤場景與對應的錯誤處理機制。
  6. 實作驗證 — 提供可執行的檢查清單，讓你驗證自己的理解是否正確。

適合對象

本教學適合以下讀者：

  • 初學者 — 想了解 Claude Code 內部架構全貌，建立系統性認知。
  • 中階開發者 — 已使用過 Claude Code，想深入理解啟動流程、工具系統與權限機制的運作細節。
  • 架構導向讀者 — 關注設計取捨、狀態管理與擴充機制，希望從原始碼層級掌握系統設計。

無論你的經驗程度如何，都能透過 L1／L2／L3 多層次設計找到合適的閱讀深度。

快速起步

先建立整體心智圖，再進入主流程。本章以鳥瞰視角介紹 Claude Code 的核心模組與彼此的協作關係，幫助你在 90 秒內掌握系統全貌。適合作為所有章節的起點。

2

啟動流程

從 CLI 入口到互動模式的完整啟動路徑。本章拆解指令列參數解析、環境初始化、模式分流（REPL／單次執行／管線模式）的每一步驟，讓你理解程式啟動時到底發生了什麼。

前往
3

指令系統

命令如何承載語義，並與工具系統協作。本章解析 slash command 的註冊、解析與分派機制，說明指令如何轉換為具體的工具呼叫與回應流程。

前往
4

工具系統

Tool 契約、工具集組裝與閘控機制。本章深入探討工具的定義介面、參數驗證規則與權限閘控流程，並說明系統如何動態組裝可用工具清單提供給模型使用。

前往
5

回合引擎

從 submit 到收尾的完整回合生命週期剖析。本章追蹤一次使用者輸入如何觸發 API 呼叫、工具執行與串流回應，並解析重試、中斷與錯誤復原等邊界情境。

前往
6

服務與 MCP

外部依賴整合邊界與能力映射機制。本章說明 Claude Code 如何透過 Model Context Protocol 連接外部服務，並解析服務發現、能力宣告與錯誤處理的設計。

前往
7

狀態與任務

狀態欄位如何驅動互動行為的實例分析。本章以 PromptSuggestion 為切入點，展示狀態機的設計模式，並說明任務佇列如何管理多步驟操作的執行順序與相依關係。

前往
8

模式與權限

工具操作前的安全決策流程。本章解析權限模式（Plan／Auto Accept／預設模式）的切換邏輯，並追蹤每一次工具呼叫如何經過權限閘門的許可或拒絕判斷。

前往
9

記憶系統

Session、Durable、Team 三層記憶機制的完整剖析。本章說明各層記憶的儲存範圍、載入時機與合併策略，並解析 CLAUDE.md 檔案如何成為持久記憶的載體。

前往
10

代理與擴充

委派工作與擴充能力的安全方法。本章介紹子代理（sub-agent）的委派機制、MCP 擴充工具的載入方式，以及如何在保持安全邊界的前提下擴展系統能力。

前往
11

提示系統

提示不是單一文字區塊，而是分層控制面。本章解析系統提示組裝、工具提示、側查詢提示與代理提示的八大家族，並說明如何撰寫可靠的提示與避免常見失敗模式。

前往
12

Hooks 與自動化

事件驅動的行為擴充機制。本章說明如何透過 Hook 在工具呼叫、訊息送出等事件前後自動執行 shell 指令、LLM 提示、HTTP 請求或代理驗證，以及事件-匹配器-掛鉤的三層管線設計。

前往
13

終端 UI

用 React 渲染終端機介面的完整框架。本章拆解自建 React 調和器、Yoga Flexbox 排版引擎、雙緩衝與 Blit 優化、2D Cell 螢幕模型等核心技術，理解 CLI 如何實現瀏覽器等級的渲染能力。
