Claude Code 初學者架構教學

本教學採三層資訊架構：首頁先選模組，再進入主題群組頁，最後閱讀各章節的由淺入深內容。

三種閱讀入口

同一章可以用不同方式讀，不需要被固定層級格式綁住。

先抓全貌

用最短時間掌握責任邊界與主流程，建立共同語言。

沿著執行路徑看

聚焦關鍵判斷、狀態轉換與常見改動點，直接對應實作。

深入取捨與風險

對照替代方案與失敗路徑，建立可維護的設計判斷。

章節閱讀骨架

所有章節遵循同一個閱讀順序，確保你在不同主題下也能維持一致節奏。

  1. 90 秒總覽：先定義這章解決的核心問題。
  2. 執行路徑：用流程圖與片段程式碼掌握主幹。
  3. 關鍵決策：理解設計選擇與代價。
  4. 替代方案取捨：知道為什麼不是另一種做法。
  5. 失敗路徑：先看錯誤，再看防護。
  6. 實作驗證：用最小驗證集確認你真的看懂。

四階段學習路線

建議按階段前進，先看總覽，再依需求深入執行細節與取捨分析。

階段一：基礎概念快速起步 → 啟動流程 → 指令系統
階段二：核心機制Skills 機制 → 工具系統 → 回合引擎 → 服務與 MCP
階段三：進階架構狀態與任務 → 模式與權限 → 記憶系統 → 代理與擴充
階段四：深入主題提示系統 → Hooks 與自動化 → 終端 UI

依角色推薦閱讀路線

若你時間有限，可先依角色挑選章節，再回頭補完整路線。

角色 建議先讀 接著補讀 目標成果
第一次接觸 Claude Code 快速起步 → 啟動流程 指令系統 → 工具系統 能說清楚系統從輸入到回應的主流程
日常使用者（想提升效率） 指令系統 → 回合引擎 → 模式與權限 Hooks 與自動化 → 提示系統 能設計穩定的工作流並避免常見誤用
維護者／架構導向讀者 工具系統 → 狀態與任務 → 記憶系統 代理與擴充 → 服務與 MCP → 終端 UI 能評估設計取捨與擴充邊界

常見閱讀問題

我一定要照順序讀完 13 章嗎？

不用。先看首頁的角色路線快速上手，再補四階段路線，效率最高。

每章都要從頭看到尾嗎？

不必。建議先看總覽建立地圖，遇到工作需求時再下鑽細節段落。

我想直接改程式，先看哪裡？

先看「啟動流程、回合引擎、模式與權限」，這三章最能降低改動風險。

建議學習節奏（7 天）

這是一個可直接執行的最小學習計畫，適合想在一週內建立穩定心智模型的讀者。

Day 1-2快速起步、啟動流程、指令系統（先看總覽與主流程）
Day 3-4工具系統、回合引擎、模式與權限（補齊失敗路徑）
Day 5記憶系統、提示系統（聚焦穩定性與一致性）
Day 6-7Hooks 與自動化、代理與擴充、終端 UI（看擴充邊界）

閱讀建議：先從「快速起步」建立全貌，再按四階段推進。每章先看總覽，再視需求深入細節與取捨。
