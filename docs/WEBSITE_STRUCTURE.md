# Website Structure Specification

This document defines the website structure contract for the tutorial site.

## Purpose

- Keep navigation behavior consistent across pages.
- Keep ownership boundaries explicit between HTML, CSS, and shared script logic.
- Ensure structure changes are verifiable and documented.

## Scope

- Public tutorial pages only: `index.html` and `pages/*.html`
- Shared website assets under `assets/`
- Navigation, breadcrumb, pager, and page skeleton contracts

## Page Types

- Home page: `index.html`
  - Acts as the global entry page.
  - Presents module-group entry cards.
- Group pages: `pages/group-*.html`
  - Aggregate chapter pages within one module boundary.
  - Provide chapter ordering and local wayfinding.
- Chapter pages: `pages/*.html` (non-group pages)
  - Hold topic details and deep technical walkthrough content.

## Directory and Asset Layout

- `index.html`: home page
- `pages/group-*.html`: group pages
- `pages/*.html`: chapter pages
- `assets/site.css`: stylesheet composition entrypoint
- `assets/styles/tokens.css`: design tokens and theme variables
- `assets/styles/layout.css`: layout shell and responsive structure
- `assets/styles/components.css`: reusable component styling
- `assets/site.js`: shared navigation and page behavior

## Source of Truth Rules

- Global navigation links MUST be defined in `assets/site.js` (`NAV_ITEMS`).
- Group-to-chapter relationships MUST be defined in `assets/site.js` mappings.
- Individual pages MUST NOT hardcode a different global navigation model.

## Page Skeleton Contract

Every website page MUST include:

1. `<body data-page="...">` with a stable page key
2. `.shell` wrapper
3. `.header` with title row and `.top-nav`
4. main content container (`<main>`)
5. shared script include (`assets/site.js` or `../assets/site.js`)

Chapter pages SHOULD include:

- `.breadcrumb` near the header,
- `.pager` near page bottom.

Both may be generated by `assets/site.js`.

## Navigation Contract

- `.top-nav` MUST be rendered from `NAV_ITEMS` in `assets/site.js`.
- `NAV_ITEMS` MUST contain only home and group entries.
- Chapter pages MUST NOT appear as top-level nav items.
- Active top-nav state MUST resolve to the current group when viewing a chapter page.

## Breadcrumb and Pager Contract

- Chapter breadcrumb format MUST be: `首頁 / 群組頁 / 當前章節`.
- Chapter pager MUST prioritize within-group movement.
- Chapter pager MUST provide a direct path back to its group page.
- Cross-group linear chapter chaining SHOULD be avoided.

## Styling Boundaries

- `tokens.css`: theme tokens and primitives only
- `layout.css`: page-level layout and responsive behavior only
- `components.css`: component presentation only
- `site.css` MUST remain a thin composition layer

## Path Rules

- Root page uses `assets/*` paths.
- Pages under `pages/` use `../assets/*` paths.
- `site.js` owns shared navigation path generation.

## Change Procedure

When adding a new group page:

1. create `pages/group-*.html`
2. assign a unique `data-page` key
3. add one `NAV_ITEMS` entry in `assets/site.js`
4. add/update group mapping in `assets/site.js`
5. update this document in the same change set

When adding a new chapter page:

1. create `pages/*.html` (non-group)
2. assign a unique `data-page` key
3. attach it to exactly one group mapping in `assets/site.js`
4. ensure breadcrumb/pager resolve correctly
5. update this document in the same change set

## Verification Checklist

At minimum, verify:

1. every page has `body[data-page]`
2. `.top-nav` renders correctly on home, group, and chapter pages
3. chapter pages highlight the correct group in top-nav
4. chapter breadcrumb links to the correct group page
5. chapter pager stays in-group and provides "回群組"
6. mobile layout (`<= 760px`) has no overflow
7. Mermaid loads only when `.mermaid` exists

## Documentation Sync Policy

- Any structural website change MUST update this file in the same change set.
- If repository-level architecture docs reference website structure, keep `docs/CODEBASE_STRUCTURE.md` aligned.
